## 5. 当前风险与 gate 条件（P0）

### 风险 R1：3756 场景下重定位求解失败导致中断或退化

- 现象：`gurobi_reposition_policy` 在高压力下可能出现 `no incumbent` / 异常路径
- 风险：整轮实验失真或中断，影响结论可信度

### 风险 R2：`charging_deadline_miss_ratio` 可能被分母放大

- 定义：`charging_deadline_misses / successful_swaps`
- 当 `successful_swaps` 偏小且 `charging_deadline_misses` 偏高时，比值会异常放大

### 通过 gate 的最低条件

1. 诊断 episode 必须完整运行并输出求解状态日志  
2. 明确区分：求解失败导致的策略退化 vs 指标定义导致的比值放大  
3. 在 gate 通过前，不启动 162-run 全量矩阵
